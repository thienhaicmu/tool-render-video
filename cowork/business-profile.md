# Business Profile — Render Studio

## What This Tool Does

Render Studio is a local desktop tool for a single operator who:
- Downloads long-form YouTube videos or uses local video files
- Automatically cuts them into short-form viral clips (TikTok / Reels length)
- Burns subtitles, applies color grading, adds text overlays
- Uploads finished clips to TikTok on a schedule

The entire pipeline runs on one Windows machine. There is no cloud processing, no subscription, no external API required for the core workflow.

## Operator Profile

- Single operator managing multiple TikTok channels
- Each channel has its own folder, account profile, proxy config, and output directory
- Operator runs render jobs during the day and schedules uploads for peak times (07:00, 17:00)
- Operator monitors output quality and intervenes when needed

## Typical Daily Workflow

```
1. Open desktop app
2. Select channel
3. Paste YouTube URL → Download & Edit
4. Preview video in editor, set trim points, configure subtitles and text overlays
5. Click Render → monitor parallel part rendering
6. Review output in channels/<code>/video_out/
7. Upload → schedule → monitor TikTok upload
```

## Channel Setup

Each channel lives at `channels/<code>/` with:
- `video_out/` — render output
- `upload/source|uploaded|failed/` — upload tracking
- `account/upload_settings.json` — channel-level defaults
- `hashtag/hashtags.txt` — auto-appended to captions
- `logs/` — per-job render logs

## Data That Must Not Be Lost

| Data | Location | Why |
|---|---|---|
| Channel configs | `channels/<code>/account/` | Account profiles, proxy, browser state |
| Render output | `channels/<code>/video_out/` | Finished videos ready for upload |
| Upload records | `channels/<code>/upload/uploaded/` | Proof of upload, avoids re-upload |
| Job database | `data/app.db` | Job history, resume state |
| Render reports | `data/reports/` | Per-job Excel reports with scores |

## What Can Be Regenerated

- Preview sessions (`data/temp/preview/`) — recreated by running prepare-source again
- Subtitle `.srt`/`.ass` files — regenerated by re-running the render
- Whisper models (`data/whisper_cache/`) — re-downloaded on next warmup
- Viral model (`data/viral_model.pkl`) — retrained when enough feedback is recorded

## Performance Constraints

- Render speed is bounded by CPU/GPU and the adaptive worker cap
- NVENC (GPU encode) is significantly faster than CPU for long render jobs
- Whisper transcription is the bottleneck on CPU-only machines
- Preview transcode for long videos (>30 min) can take 2–5 minutes

## Key Quality Signals

- `viral_score` — heuristic score per segment, higher = stronger content hook
- `output_fps` — always capped to source FPS (never upscaled)
- `motion_aware_crop` — recommended on; uses OpenCV tracking for subject-centered crop
- `render_profile` — `balanced` for daily use, `quality` or `best` for hero content
