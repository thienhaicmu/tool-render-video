# Task Execution Policy

## Steps
1. Read `doc/project-context.md` and `doc/engineering-standards.md`.
2. Restate task, scope, constraints, assumptions.
3. Make a short plan.
4. Implement minimal in-scope changes.
5. Validate changed behavior.
6. Return structured result.

## Required Output
- `## Summary`
- `## Assumptions`
- `## Plan`
- `## Changes Made`
- `## Verification`
- `## Risks / Follow-ups`
