### 1.3 The architectural gap

Today Recap path: Whisper → Comprehension (StoryModel) → Editorial → Pass-3 binding (with story+editorial in prompt context).
Today Clip path: Whisper → segment selection (NO StoryModel context).

**The gap:** Clip's `select_render_plan` LLM call doesn't know:
- Who the characters are (could be cited in clip titles / hooks)
- What the central conflict is (could drive `viral_score` weighting)
- What the emotional curve is (could shape `hook_intensity` selection)
- What the theme is (could ground clip ranking in narrative resonance)

Closing this gap is what the architecture review called the **#1 strategic
priority**.

## 2. Replacement design — mirror Batch C's recap wiring

### 2.1 Schema layer (Phase 1 — substrate only)

**New field on `RenderRequest`** ([models/render.py](../backend/app/models/render.py)):

```python
# Architecture-review C.1 Phase 1 (2026-06-30): when True AND render_format
# is "clips", the Comprehension stage runs on the Whisper transcript to
# produce a StoryModel, which is then passed to select_render_plan so the
# LLM grounds clip selection in whole-film semantic understanding (theme,
# conflict, characters, emotional curve). Default False — Sacred Contract
# #2: stored historical payloads replay bit-identically with the legacy
# no-StoryModel path. Effective only on the Clip path; Recap always runs
# Comprehension (gated by STORY_INTELLIGENCE_HOIST_ENABLED env var since
# Batch C).
use_story_intelligence: bool = False
```

**No validator needed.** Pydantic's default `False` handles missing field on
deserialise. `extra="ignore"` on RenderRequest swallows any legacy
field that gets dropped.

### 2.2 Pipeline layer (Phase 2 — wire-in, CRITICAL)

In `render_pipeline.py` just before [line 888](../backend/app/features/render/engine/pipeline/render_pipeline.py#L888):

```python
# Architecture-review C.1 Phase 2 (TBD): when use_story_intelligence is True,
# run the Comprehension stage on the Whisper transcript to produce a
# StoryModel, then thread it into select_render_plan so the LLM grounds
# clip selection in whole-film semantic understanding. Sacred Contract #3
# spirit: Comprehension failure → story_model stays None → select_render_plan
# runs identically to the legacy path.
_story_model = None
if getattr(payload, "use_story_intelligence", False):
    try:
        from app.features.render.engine.pipeline.comprehension_stage import (
            run_comprehension as _run_comprehension,
            is_hoist_enabled as _comprehension_enabled,
        )
        if _comprehension_enabled():
            _story_model = _run_comprehension(
                job_id=job_id, channel_code=effective_channel,
                srt_content=_ai_srt_content,
                video_duration=_ai_video_duration,
                provider=_ai_provider, api_key=_ai_api_key,
                model=getattr(payload, "llm_model", None),
                target_language=getattr(payload, "voice_language", "vi-VN"),
                tone=getattr(payload, "rewrite_tone", ""),
                emit_fn=_emit_render_event,
            )
    except Exception:
        _story_model = None  # Sacred Contract #3 — never abort the render

_render_plan = _llm_select_render_plan(
    ...,
    story_model=_story_model,  # NEW kwarg
)
```

### 2.3 Provider layer (Phase 3 — actual closure)

Each of the 3 providers gets an additional optional kwarg:

```python
# ai/llm/providers/gemini.py:select_render_plan signature
def select_render_plan(
    srt_content: str,
    output_count: int,
    ...,
    story_model: Optional["StoryModel"] = None,  # NEW
) -> Optional[RenderPlan]:
```

Inside each provider, when `story_model is not None`:
1. Render a `_story_block(story_model)` text section (reuse the Batch C
   helper from `recap_prompts._story_block` — it already exists and is
   prompt-tested)
2. Inject the block into the clips user prompt via a new template slot

The dispatcher in [`ai/llm/__init__.py`](../backend/app/features/render/ai/llm/__init__.py):

```python
def select_render_plan(
    *,
    provider: str = DEFAULT_PROVIDER,
    srt_content: str,
    ...,
    story_model: Optional[Any] = None,  # NEW
):
    ...
    kwargs = dict(..., story_model=story_model)
    ...
```

Same dispatcher shape as `select_recap_plan` already has from Batch C.

### 2.4 Prompt + PROMPT_VERSION (Phase 3)

The clips user prompt template (`ai/llm/prompts.py`) gets a new optional
section that's empty when `story_model is None`:

```
{story_block}
{...rest of template...}
```

When `story_model` is non-None, `_story_block(story_model)` (already exists
in `recap_prompts.py`) fills it with the canonical block format.

`PROMPT_VERSION = 1 → 2` bump in `prompts.py` because the wire shape of
the user prompt now has a `{story_block}` slot that didn't exist before.
This invalidates the entire LLM disk cache by construction — one-render
warm-up cost; acceptable.

## 4. Decision tree

```
render_pipeline.py reaches LLM call site
└─ payload.use_story_intelligence == True?
   ├─ NO  → _llm_select_render_plan(..., story_model=None)
   │         (legacy path, bit-identical pre-C.1)
   └─ YES → STORY_INTELLIGENCE_HOIST_ENABLED env var ON?
            ├─ NO  → story_model=None, fall back to legacy LLM call
            │         (operator-level kill switch from Batch C respected)
            └─ YES → run_comprehension(...)
                      ├─ produced StoryModel → pass to select_render_plan
                      │   ├─ provider has story_model kwarg (Phase 3) → injected into prompt
                      │   └─ provider doesn't (Phase 1/2) → kwarg ignored (no-op)
                      └─ Comprehension failed → story_model=None → legacy path
```
