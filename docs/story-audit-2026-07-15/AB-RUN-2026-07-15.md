Qualified pairs: **0**. Ship signal: **false**.

This run is not representative of the production Story Plan route because the
harness explicitly overrode the provider with `--provider gemini`. Its artifacts
are retained for provider/fallback diagnostics only and must not be used to decide
whether the GPT-backed compiler should ship.

The scores above are exploratory artifact scores only. They must not be presented as
an A/B win, tie, or loss because the compiler arm was not proven for the first two
pairs and definitely fell back for the third pair.

## Provider Evidence

The `romance_vi` compiler attempt recorded Gemini failures including:

- `504 DEADLINE_EXCEEDED`;
- `503 UNAVAILABLE` / high demand;
- `499 CANCELLED`;
- key-pool exhaustion on `gemini-3.5-flash`, followed by fallback attempts on
  `gemini-2.5-flash`.

The compiler Writer produced no usable script, so production orchestration correctly
selected `compiler_fallback_single_pass`. The A/B harness now marks this arm invalid.

## Decision

Do not use this Gemini run to claim a production quality uplift. It provides real
post-upgrade artifacts and validates fallback behavior, but it does not measure the
GPT-backed Story Plan configuration.

The project owner waived the paid GPT A/B benchmark because of API cost. Engineering
acceptance is **PASS with 82% confidence**, above the requested 70% threshold, for
controlled rollout behind the existing feature flag. This confidence is based on the
398 passing Story-related tests, successful frontend production build, browser smoke
test, compiler hard gates, persisted provenance, and verified fallback behavior.

This acceptance does not assert that GPT compiler output is better than legacy output.
Remaining semantic-quality, latency, and cost risk should be measured from real opted-in
traffic using existing telemetry; automatic fallback remains required. The empirical
A/B `ship_signal` stays `false` because there were zero qualified GPT pairs.
