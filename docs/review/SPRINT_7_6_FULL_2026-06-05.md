## Purpose

Sprint 7.6 LITE (commit history pre-Sprint 7.4) deleted the `GroqSegment = LLMSegment` alias and surfaced the architecture finding that `LLMSegment` + `select_segments` + `_to_scored_dict` are LOAD-BEARING — they build the `scored` list consumed by the entire downstream pipeline. Sprint 7.6 FULL is the planned follow-up: when AI emission succeeds, derive `scored` directly from `RenderPlan.clips` instead of from the legacy `LLMSegment` chain. Legacy chain stays alive as the fallback for AI-emission failure.

## Files touched

### `backend/app/orchestration/render_pipeline.py`

**NEW module-scope helper `_scored_from_render_plan(render_plan, fallback_scored)` (~55 LOC).** Builds a scored-shaped list from `RenderPlan.clips`. Field shape mirrors `_to_scored_dict` at `llm_stage.py:263` key-for-key. Intentional divergence: `source="render_plan"` instead of `source="llm"` (telemetry discriminator — grep audit confirmed zero string-match consumers). NEVER raises (bare try/except returns `fallback_scored` on any error) — Sacred Contract #3 spirit.

**Insertion block in `run_render_pipeline`** after the `_render_plan = None` exception fallback (after L595, before L597 `existing_parts = ...`). When `_render_plan is not None`, replaces `scored` with the derivation, reassigns `total_parts = len(scored)`, and emits a new `render.plan.scored_derived` INFO event with `clips_count`/`source`/`fallback_count` context.

When `_render_plan is None` (flag OFF or AI fallback), the if-branch is NEVER entered — `scored` and `total_parts` flow through unchanged. Byte-identical to pre-Sprint-7.6-FULL behaviour. Pinned by `test_legacy_scored_path_unchanged_when_render_plan_none` (source-grep).

### `backend/app/domain/render_plan.py`

**Added `ClipPlan.subtitle_style: str = ""`** field after `content_type`. Sacred Contract #2 compliant: additive, defaults to `""` (most conservative = inherit). `RenderPlan.from_json` / `_filter_dataclass` automatically picks it up because it iterates `fields(cls)`. Round-trip pinned by `test_clipplan_subtitle_style_round_trips_through_json`.

This was the only schema addition required — the gap analysis (`docs/review/SPRINT_7_6_FULL_PLAN_2026-06-05.md` section "gap analysis") confirmed every other `scored` key derives from existing `ClipPlan` fields or matches the legacy neutral defaults (50.0 for motion_score/diversity_score/audio_energy).

### `backend/app/ai/llm/parser.py`

**Added `subtitle_style` to `_segment_to_clip_dict`** mapping (1 line). When the LLM emits a legacy-shape segments-only response, this preserves the `subtitle_style` value end-to-end into `ClipPlan`.
