## 3. Findings ranked by severity

### P0 (immediate)

1. **`routes/devtools.py` RCE behind single env flag + no defensive coupling against `HOST=0.0.0.0`** — `main.py:109` is the only gate; `services/dev_commands.py:1465` spawns PowerShell with `ExecutionPolicy Bypass`.
2. **Parallel WebSocket implementation in `MonitorStep.tsx:596`** — bypasses canonical `RenderSocketClient`; hardcoded `ws://127.0.0.1:8000`.
3. **`editStore.ts` not exported from `stores/index.ts:1-9` but actively imported** — architectural drift, future refactor will silently break studio flow.

### P1

1. Job manager `shutdown_job_manager(wait=False)` — abandons in-flight renders on SIGTERM (`main.py:263`, `job_manager.py:228`).
2. `RenderRequest` 14+ boolean fields default `True` — Sacred Contract 2 violation; replayed historical jobs silently activate features.
3. `jobs` table has zero indexes — full table scans on every history fetch (`connection.py:164-178`; queried at `jobs_repo.py:118`).
4. Dead frontend feature directories shipped in build (`features/premium-mockup/` empty; `features/quality/`, `features/progress/` zero inbound imports).
5. Type drift risk — `frontend/src/types/api.ts` is hand-mirrored from Pydantic with no codegen enforcement.
6. Parallel render flows reachable — `features/studio/` (older 6-step) AND `features/clip-studio/` (current 4-step) both wired in `App.tsx:26-39`.
7. WS type guard at `events.ts:36` only checks `'job' in msg`; doesn't enforce `parts`/`summary` presence.
8. No CI / `.github/workflows/` — CLAUDE.md "full pytest before CRITICAL edits" rule is unenforced.

### P2

1. NVENC semaphore not enforced uniformly — only acquired at `base_clip_renderer.py:92` and `legacy_renderer.py:266`; other FFmpeg paths bypass.
2. Cloud LLM providers (Groq, Gemini, OpenAI) lack explicit timeouts and circuit breakers.
3. `_JOB_LOG_DIRS` dict mutation unlocked (`render_events.py:17`); never pruned.
4. `_default_fallback_db_path()` silent fallback erodes Contract 7.
5. `GET /api/jobs/{id}` returns untyped dict — no `response_model=` (`routes/jobs.py:359-366`).
6. Three DB connection models in active use (CLAUDE.md says two): `db_conn()` ctx, `_thread_conn()`, raw `get_conn()` with manual close (the latter sometimes outside try/finally in `feedback_repo.py` and `creator_repo.py`).
7. No automatic cache pruning despite cache root migration (`services/maintenance.py` doesn't touch `APP_DATA_DIR/cache`).
8. No `React.memo` anywhere; all 4 step screens always mounted in `RenderWorkflow.tsx`.
9. Single global `ErrorBoundary` (`App.tsx:49-64`); no per-route boundaries.
10. Subtitle style variant maps duplicated 3× in `StepConfigure.tsx:17-76,167-180,232-247`.

### P3

Minor: `JobStage.CANCELLED` not in enum; `_drop_upload_tables` precedent; `check_perf.py` raw connect; path alias `@/*` unused; orphan modules `api/render-v2.ts`, `api/download.ts`, `data/fallbacks.ts`; inline hex colors bypassing design tokens; `_thread_conn` ping-test on every call.

## 6. Notes for future audits

- Verify line counts via `wc -l` before reasoning about file sizes. CLAUDE.md drift accumulates fast after refactors.
- Three modules now sit at the top of the blast-radius pile: `render_pipeline.py`, `part_renderer.py`, `motion_crop.py`. Decomposing them is the highest-leverage architectural investment.
- The Groq-only pipeline (`groq_only_pipeline.py`) is the sole pre-render path — there is no fallback. Audit cloud failure modes carefully if Groq SLAs become an issue.
- `data/app.db` has no backup mechanism. A single SQLite corruption event = total job-history loss. P1 candidate for Sprint 6.
